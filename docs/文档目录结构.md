# 项目交付文档目录结构（目录 / TOC 模板）

说明：下列为每种交付文档的建议目录结构（章节与子项）。乙方可据此生成最终文档，便于满足 `技术要求.md` 和 `合同.md` 的文档完整性条款。

1) 需求分析报告
- 1. 封面（项目名称、版本、编写人、日期）
- 2. 修订记录
- 3. 术语与缩写
- 4. 项目概述（背景、目标、范围）
- 5. 业务流程描述
  - 5.1 关键业务流程图（流程图/泳道图）
  - 5.2 用例列表与说明
- 6. 功能需求
  - 6.1 用例/功能清单（编号）
  - 6.2 每项功能的详细说明：输入、处理、输出、边界条件
  - 6.3 非功能性需求（性能、并发、可用性、安全、兼容性）
- 7. 数据需求（主要数据项、数据字典概览）
- 8. 接口需求（外部系统、API、文件格式）
- 9. 业务规则与权限控制
- 10. 可追溯性矩阵（需求→用例→测试用例）
- 11. 验收标准与验收测试计划
- 12. 风险分析与缓解措施
- 13. 附件（样例病历、数据样本、相关合同节选）

2) 软件架构设计说明
- 1. 封面与修订记录
- 2. 文档目的与范围
- 3. 架构概述（目标、约束、关键场景）
- 4. 总体架构图（组件图、部署图）
- 5. 子系统与模块说明
  - 5.1 模块职责与接口说明
  - 5.2 数据流与控制流
- 6. 数据存储方案（数据库、文件、向量库）
- 7. 模型服务与推理架构
  - 7.1 模型清单与用途（embedding、rerank、LLM）
  - 7.2 推理框架与性能优化（vLLM、显存优化、并发策略）
  - 7.3 模型部署（容器/镜像、硬件需求）
- 8. 检索与RAG引擎设计（分块、K/N 配置、重排序流程）
- 9. 接口/API 设计（REST/OpenAI兼容接口、鉴权、吞吐）
- 10. 安全与隔离设计（双网卡隔离、TLS、权限分级）
- 11. 监控与运维（指标、告警、日志采集）
- 12. 扩展性和可维护性说明
- 13. 部署方案与回滚策略
- 14. 非功能性需求映射（性能、可靠性、兼容性验证点）
- 15. 附录（第三方依赖、配置示例）

3) 数据库设计方案
- 1. 封面与修订记录
- 2. 设计概述（目标、范围、约束）
- 3. 数据模型总览（ER图/概念模型）
- 4. 逻辑模型（表/集合 列表）
  - 4.1 表结构定义（字段、类型、约束、默认值）
  - 4.2 索引策略与说明
  - 4.3 外键与参照完整性
- 5. 向量存储设计（向量表/向量库、维度、分片策略）
- 6. 数据分区、归档与生命周期策略
- 7. 数据字典（字段含义、来源、示例）
- 8. 事务与并发控制策略
- 9. 备份与恢复策略（RTO/RPO 指标）
- 10. 安全与权限（数据加密、行/列级加密、权限分配）
- 11. 数据迁移方案（从现有系统导入）
- 12. 性能调优（索引、查询优化、批量导入方案）
- 13. 接口与导出（API、导入导出格式）
- 14. 附录（DDL 脚本样例、ER 图文件）

4) 单元测试报告
- 1. 封面与修订记录
- 2. 测试范围与目标（包含哪些模块/包）
- 3. 测试环境（OS、Python 版本、依赖清单）
- 4. 测试方法与工具（pytest、coverage 等）
- 5. 测试用例汇总（按模块分类）
  - 5.1 用例编号、目的、前置条件、步骤、预期结果、实际结果、状态
- 6. 覆盖率报告
  - 6.1 行覆盖率/分支覆盖率统计
  - 6.2 未覆盖代码片段说明与风险评估
- 7. 测试执行记录（时间、执行人、日志路径）
- 8. 缺陷统计与修复状态
- 9. 结论与风险提示
- 10. 附件（测试脚本、关键日志片段）

5) 集成测试报告
- 1. 封面与修订记录
- 2. 测试范围与目标（接口、子系统集成、端到端流程）
- 3. 测试环境（部署拓扑、依赖服务、测试数据说明）
- 4. 测试用例与场景（接口调用序列、端到端场景）
  - 4.1 每个场景的步骤、期望输出
- 5. 数据准备与清理策略
- 6. 性能/并发测试结果（TTFT、并发数、吞吐）
- 7. 稳定性测试（72 小时运行日志、异常记录）
- 8. 安全性测试（权限、鉴权、输入校验）
- 9. 缺陷统计与修复跟踪
- 10. 验收测试结果（映射至合同验收条款）
- 11. 结论与建议
- 12. 附件（接口抓包、日志、测试脚本）

6) 开发进度报告（周期性交付）
- 1. 封面与版本信息
- 2. 本期摘要（已完成、进行中、下阶段）
- 3. 里程碑与甘特图
- 4. 需求变更记录
- 5. 风险与阻塞项
- 6. 代码提交统计（git 提交、主要分支、tag）
- 7. 测试与质量指标（覆盖率、已修复缺陷数）
- 8. 部署/验收计划
- 9. 人员与现场服务安排（现场服务天数统计）
- 10. 下期计划与资源需求
- 11. 附件（会议纪要、评审记录）

7) 《软件测试报告》（验收用，含 500 条测试用例）
- 1. 封面与修订记录
- 2. 测试目标与范围（覆盖 500 条测试用例说明）
- 3. 测试环境与准备
- 4. 测试用例清单（附索引）
  - 4.1 按功能分组列出用例编号与概要
  - 4.2 每条用例的完整执行记录（前置、步骤、输入、期望、实际、截图/日志）
- 5. 自动化与手工用例说明
- 6. 执行统计（通过率 100% 的证明与日志）
- 7. 覆盖映射（用例→需求→验收条款）
- 8. 问题与偏差说明（若有，应记录并标注已关闭证明）
- 9. 结论与签章（测试负责人签名）
- 10. 附件（执行日志、关键截图、测试脚本）

8) 《用户操作手册》
- 1. 封面与版本信息
- 2. 介绍（系统概述、目标用户、术语表）
- 3. 快速入门（安装/访问、登录、基础配置）
- 4. 常用操作流程（按角色/场景组织）
  - 4.1 医学科研场景操作流程图（示例流程图）
  - 4.2 病例录入流程
  - 4.3 报告生成与导出流程
  - 4.4 检索与证据溯源操作
- 5. 高级功能指南（模型选择、参数调整、批量导入）
- 6. 故障排查与常见问题（FAQ）
- 7. 账户与权限管理操作
- 8. 附件（术语、参考表、常用命令）

9) 《数据安全维护手册》
- 1. 封面与修订记录
- 2. 安全策略总览（法律法规与公司策略对应）
- 3. 数据分类与保护需求
- 4. 存储与传输加密策略（算法、密钥管理）
- 5. 访问控制与最小权限策略（用户/角色/审计）
- 6. 日志管理与审计（日志保留策略、审计流程）
- 7. 备份与恢复（备份频率、存储位置、恢复演练）
- 8. 应急响应流程（事件识别、通报、处置、恢复）
- 9. 安全测试与合规检查（渗透测试、第三方合规）
- 10. 运维与更新策略（补丁管理、配置变更）
- 11. 培训与交付（交付清单、培训计划、操作演示）
- 12. 附件（密钥管理流程、样例演练记录）
